# Model Recording Analysis

## Overview

This folder contains detailed statistical analyses of the predictions generated by all evaluated deep learning models on the complete test dataset.

Unlike the evaluation tables, which summarize overall model performance, the files in this folder provide a recording-level analysis of the prediction results. They were used to identify common failure patterns, compare model behavior across different spoofing attack types, evaluate computational efficiency, and support the final model selection.

---

## Files Description

### 01_model_error_summary.csv

Provides an overall performance summary for each evaluated model.

For every model, the table reports:
- Total number of evaluated recordings
- Number of correctly classified recordings
- Number of misclassified recordings
- Error rate
- Overall accuracy

This table provides a quick comparison of the overall detection performance of all evaluated models.

---

### 02_errors_by_attack_type.csv

Summarizes the number of classification errors for each spoofing attack type.

For every attack category, the table reports:
- Number of errors produced by each model
- Total number of errors across all models

This analysis helps identify which spoofing techniques are more difficult to detect and compares the robustness of each model against different attack types.

---

### 03_recordings_by_number_of_failed_models.csv

Contains a detailed analysis of every recording in the test dataset.

For each recording, the table includes:
- Recording identifier
- Attack type
- Ground-truth label
- Prediction correctness for each evaluated model
- Number of models that failed
- Number of models that classified the recording correctly
- List of failed models
- List of successful models

This file enables recording-level error analysis and identifies recordings that are consistently difficult to classify.

---

### 04_recordings_majority_models_failed.csv

Contains only recordings that were misclassified by the majority of the evaluated models.

These recordings represent particularly challenging samples that expose common weaknesses shared across multiple architectures.

---

### 05_recordings_all_models_failed.csv

Contains recordings that were misclassified by every evaluated model.

These samples represent the most difficult recordings in the dataset and may indicate ambiguous cases, challenging spoofing attacks, or potential limitations of the available data.

---

### 06_efficiency_summary_all_models.csv

Summarizes the computational efficiency of each evaluated model.

The reported measurements include:
- Model file size
- Number of parameters
- Memory usage
- RAM consumption
- Peak GPU memory
- Average inference time per batch
- Average inference time per recording
- Input representation
- Batch size
- Number of evaluated recordings
- Number of classification errors

This table supports comparison of computational requirements in addition to classification performance.

---

### 07_final_presentation_summary.csv

Provides a consolidated summary of the most important evaluation results used during the final comparison and presentation.

The table combines:
- Classification performance
- Error statistics
- Model size
- Number of parameters
- Memory consumption
- Inference speed
- GPU memory usage

This file serves as a single reference table containing the primary performance and efficiency indicators for all evaluated models.
